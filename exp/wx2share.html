# 1. 微信端分享二次分享 url 的坑(目前只在 ios 中出现, android 中分享 ok. 2018.12)

​ des: 在 ios 中进入初始 url , 切换路由后再次分享, 分享出去的路由为初始进入 url.

​ do:
所以我们的解决方案是: 当且仅当在 ios 和 微信 中, 我们在路由切换的时候重新进行签名,即可解决.

router.beforeEach((to, from, next) => {
    if(isIOS() && isWX()) {
        // TODO: 请求微信 js-sdk 配置
        post(url).then(res=> wx.config(...))
    }
  next()
})
