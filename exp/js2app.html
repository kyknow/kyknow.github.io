# js 跳转 app

# 方案:

1. Scheme url 协议跳转 (ios, android通用)
2. Universal Link 跳转( ios9 以上可使用)
3. 微信端(应用宝-微下载)

# 1. Scheme url 协议跳转

# 实现方案:

 





 









  // 1.iframe  跳转 (有兼容问题,抛弃),ios9开始不支持在iframe内调起，需要直接使用location.href = Scheme
    var ifr = document.createElement('iframe');
    ifr.src = 'weixin://';
    ifr.style.opacity = '0'
    document.body.appendChild(ifr);

  // 2. window.location.href 跳转(使用该方案)
    window.location.href = 'weixin://'
    
  // 生产方案:
  	 window.location.href = 'weixin://'
  	 setTimeout(function (){
           window.location.href = isIOS? APP STORE : ANDROID DOWNLOAD 
  	 },2000)

优点: 通用, 微信端无法使用

缺点:

IOS:

在 ios 端调用的话, 会弹出一个对话框, 打开和取消 两个按钮.点击打开, 直接跳转;点击取消, 会出现这种框

而且会阻塞我们两秒之后执行的定时器, 无法跳转 app store

ANDROID:

安卓无毛病, 正常体验. 但是在Android的某些浏览器中,基于一些安全机制,必须让用户主动点击打开/下载按钮才能触发 scheme url 跳转 app. 如果需要在页面 load 之后自动触发 scheme url,这样在某些浏览器中是行不通的.必须让用户主动触发按钮来跳转

有 APP 直接跳转 APP, 无对话框确认

无 APP 2S 后跳出下载 APP框

# 2. Universal Link (ios9 以上使用)

直接看这篇文章, 详细解释并提供了两套方案解决

# 3. 应用宝的微下载

直接在应用宝中注册,并使用微下载功能,根据他们提供的 url, 在微信端可以简介跳转.

在微信端和 QQ 端使用的原因是因为 腾讯把 scheme url 和 universal url 给禁止了.无法跳转.

除非上他们的白名单
